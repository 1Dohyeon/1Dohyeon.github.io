# 커피 생두 가격 예측 시스템

## 프로젝트 개요

- **개발 기간**: 2025.03 ~ 2025.06
- **기술 스택**: PyTorch, FastAPI, React
- **프로젝트 유형**: 6인 프로젝트 (대학 캡스톤)
- **맡은 역할**: 프론트엔드, FastAPI 서버, 기후/경제 데이터 가공, 모델 설계 및 리뷰 참여

## 배경

기후변화와 경제 불확실성으로 커피 시장의 가격 변동성이 커지면서 원두 구매 업체들이 최적의 구매 시점을 잡기 어려워지고 있습니다. 커피 재배 지역의 기후 데이터, 거시 경제 지표, 뉴스 기사 감성 분석을 결합해 단기 커피 선물 가격을 예측하는 시스템을 목표로 했습니다.

## 데이터를 어떻게 구성했는가

브라질, 콜롬비아, 에티오피아의 NASA 기후 데이터와 yfinance 기반 경제 지표를 수집했습니다. 여기서 첫 번째 문제가 생겼습니다. 기후 데이터는 일별 단위지만 거시 경제 지표는 월별 또는 연도별입니다. 해상도가 다른 데이터를 단순히 일별로 복제하면 직선적인 계단 형태가 되어 모델이 왜곡된 패턴을 학습할 수 있었습니다.

**스플라인 보간(Spline interpolation)** 으로 해결했습니다. 월/연도 단위 데이터를 자연스러운 곡선으로 변환해 일별 데이터와 함께 통합했습니다. 여기에 일간 수익률, 단기 변동성, 모멘텀 등의 파생 변수를 추가로 생성하고, 비수확기 강수량 평균처럼 계절 특성을 반영한 정적 변수도 구성했습니다.

## 왜 단기 예측으로 방향을 바꿨는가

초기 목표는 2개월 이상의 장기 가격 흐름을 예측하는 것이었습니다. **LSTM + Attention 메커니즘** 구조로 100일치 입력 시퀀스를 받아 예측을 수행했지만, 급격한 가격 변동은 모델이 포착하지 못했습니다. 상승·하락 추세는 어느 정도 맞혔지만, 변동성이 큰 구간에서 예측이 크게 틀렸습니다.

팀 논의를 거쳐 **7일과 14일 단기 예측으로 전략을 전환**했습니다. 두 모델을 병렬로 운용하면서 단기 이상 징후 탐지와 중기 추세 판단을 분리했고, 두 예측 간 차이가 클 때는 예측 불일치 경고 신호를 발생시키는 규칙 기반 레이어도 추가했습니다.

최종 성능은 7일 모델 RMSE 9.14, MAE 6.64 / 14일 모델 RMSE 12.91, MAE 8.94입니다. 예측 기간이 짧을수록 성능이 높아지는 결과로, 단기 예측으로 전환한 판단이 옳았습니다.

## 감성 분석을 어떻게 활용했는가

가격 예측 모델과 별개로 뉴스 기사 감성 분석 모델을 독립적으로 개발했습니다. 수집된 기사 제목을 **FinBERT**로 분류해 날짜별 감성 점수를 생성하고, 이를 가격 데이터와 결합해 UI에서 시각화했습니다.

단, 실제로는 90%의 기사가 중립으로 분류되어 감성 분석 결과는 메인 예측 모델이 아닌 참고 지표로만 활용됐습니다. 감성 지표가 예측 신뢰도를 높이지 못한다는 결과 자체도 의미 있는 발견이었습니다.

## 현재 한계

애초 목표는 실시간 데이터를 수집해 당일 기준 미래 가격을 예측하는 것이었습니다. 그러나 거시 경제 지표 대부분이 유료 API 또는 월별 발표 데이터라 실시간 수집에 제약이 있었습니다. 현재는 확보된 과거 데이터로 학습과 테스트를 진행했고, 실시간 수집이 가능해지는 시점에 대비해 모델 아키텍처는 유연하게 설계해뒀습니다.
