# TripWith

## 프로젝트 개요

- **개발 기간**: 2024.12 ~ 2025.02
- **기술 스택**: React, NestJS, MySQL, Docker, EC2
- **협업 인원**: 3인
- **맡은 역할**: API · ERD 설계, 백엔드(사용자 · 플래너 모듈), 프론트엔드 전체

> [서버 Repository](https://github.com/tripwith-dev/rest-nestjs) · [UI Repository](https://github.com/tripwith-dev/ui-react)

## 왜 만들었는가

여러 여행 플래너 앱을 써보면서 공통된 불편함을 발견했습니다. 일정을 표 형태로 세밀하게 짜는 화면과, 장소를 지도에서 확인하는 화면이 항상 분리되어 있었습니다. 지도에서 장소를 찾고, 다시 플래너로 돌아와 입력하는 반복이 번거로웠습니다.

**테이블 형태 일정 편집과 지도를 하나의 화면에서 동시에 보여주는** 인터페이스를 목표로 했습니다. 코로나 이후 개별 여행이 늘면서 최신 여행지 정보가 중요해졌다는 점도 고려해, 여행자 간 정보를 교류할 수 있는 커뮤니티 기능도 함께 구현했습니다.

## 테이블 UI 설계: 시간 단위를 30분으로 정한 이유

테이블형 일정 UI를 설계할 때 가장 먼저 고민한 것은 시간 단위였습니다. 15분 단위로 잘게 나누면 화면에 표시되는 칸이 너무 많아져 오히려 전체 흐름이 보이지 않았습니다. 반대로 1시간 단위는 정밀한 계획을 짜기에 너무 거칠었습니다.

30분을 선택한 이유는 여행 일정의 특성 때문입니다. 이동 시간, 식사, 관광지 방문을 현실적으로 계획할 때 30분이 가장 자연스러운 최소 단위였습니다. 빡빡하게 채우기보다 여유 있는 일정을 유도하는 UX적 의도도 있었습니다.

## 핵심 기능

Google Maps API를 활용해 지도 검색으로 장소를 추가하면 테이블 일정에 자동으로 반영되도록 연동했습니다. 완성된 플래너는 공개/비공개 설정이 가능하고, 공개된 플래너에는 다른 사용자가 좋아요와 댓글을 남길 수 있습니다. 여행지 태그 기반으로 플래너를 분류하고 커뮤니티 게시글에서도 태그로 필터링할 수 있습니다.

직접 구현한 기능은 이메일 로그인, 프로필 관리, 마이페이지 일정 관리, 지도 연동 인터페이스, 테이블 일정 작성, 공개/비공개 설정, 좋아요 · 댓글, 태그 기반 분류 8가지입니다.
