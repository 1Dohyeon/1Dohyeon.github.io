# Dijkstra (다익스트라 알고리즘)

가중치가 있는 그래프에서 특정 노드로부터 모든 노드까지의 최단 거리를 구할 때 사용합니다. 핵심은 **"현재까지 알려진 최단 거리가 가장 짧은 노드부터 탐색한다"**는 그리디 전략입니다. 우선순위 큐(min-heap)로 구현합니다.

---

## 동작 원리

- `distance`: 출발지로부터 각 노드까지의 최단 거리 기록 (초기값 INF)
- `queue`: 다음 방문 후보를 `(거리, 노드)` 형태로 관리하는 min-heap

```python
import heapq

def dijkstra(start_node):
    queue = []
    heapq.heappush(queue, (0, start_node))
    distance[start_node] = 0

    while queue:
        dist, now = heapq.heappop(queue)

        # 이미 더 짧은 경로로 처리된 노드라면 무시
        if distance[now] < dist:
            continue

        for neighbor, weight in graph[now]:
            cost = dist + weight
            if cost < distance[neighbor]:
                distance[neighbor] = cost
                heapq.heappush(queue, (cost, neighbor))
```

**`if distance[now] < dist: continue`** 이 줄이 핵심입니다. 큐에 한 노드가 여러 번 들어갈 수 있는데, 이미 더 짧은 경로로 처리된 경우 건너뜁니다.

---

## 시뮬레이션 (백준 1753 예제)

- **정점**: 1~5번 / **시작점**: 1번
- 간선: 1→2(2), 1→3(3), 2→3(4), 2→4(5), 3→4(6), 5→1(1)

| 단계 | 처리 노드 | distance 갱신 |
|---|---|---|
| 초기 | - | `[0, INF, INF, INF, INF]` |
| 1 | 1번 (dist=0) | 2번=2, 3번=3 |
| 2 | 2번 (dist=2) | 4번=7 (3번은 6 > 3이라 무시) |
| 3 | 3번 (dist=3) | 4번=9 > 7이라 무시 |
| 4 | 4번 (dist=7) | 이웃 없음, 종료 |

최종 결과: 1번=0, 2번=2, 3번=3, 4번=7, 5번=INF

---

## 정리

- 음수 가중치가 없는 그래프에서 단일 출발점 최단 경로 탐색
- 시간 복잡도: O((V + E) log V) — V는 정점 수, E는 간선 수
- `distance` 배열 + min-heap의 조합이 핵심
- 음수 가중치가 있다면 벨만-포드(Bellman-Ford)를 사용해야 합니다
