# OSI 7계층 vs TCP/IP 4계층

HTTP 요청 하나가 실제로 어떻게 전달되는지 따라가다 보면, 여러 단계를 거친다는 것을 알 수 있습니다. 이 단계를 표준화한 것이 **계층 모델**입니다. OSI 7계층과 TCP/IP 4계층, 두 모델이 어떤 역할을 하고 서로 어떻게 다른지 정리합니다.

---

## OSI 7계층

| 계층 | 이름 | 핵심 역할 | 대표 프로토콜/장비 |
|:---:|---|---|---|
| 7 | 응용 (Application) | 사용자와 직접 맞닿는 인터페이스 | HTTP, FTP, DNS, SMTP |
| 6 | 표현 (Presentation) | 데이터 형식 변환, 암호화/복호화, 압축 | SSL/TLS, JPEG, ASCII |
| 5 | 세션 (Session) | 통신 세션 수립·유지·종료 | NetBIOS, RPC |
| 4 | 전송 (Transport) | 신뢰성 있는 端對端 전송, 포트 번호 | TCP, UDP |
| 3 | 네트워크 (Network) | IP 주소 기반 경로 결정 (라우팅) | IP, ICMP, 라우터 |
| 2 | 데이터링크 (Data Link) | MAC 주소 기반 프레임 전송, 오류 감지 | Ethernet, 스위치 |
| 1 | 물리 (Physical) | 비트를 전기·광 신호로 변환 | 케이블, 허브 |

---

## 캡슐화와 역캡슐화

왜 굳이 7단계로 나눴을까요? 각 계층이 자신의 역할만 담당하고 아래 계층에 책임을 위임하는 구조이기 때문입니다. 덕분에 한 계층의 구현이 바뀌어도 나머지 계층은 영향을 받지 않습니다.

- **송신 측:** 7계층 → 1계층 방향으로 각 계층이 헤더를 추가하며 포장
- **수신 측:** 1계층 → 7계층 방향으로 각 계층이 헤더를 제거하며 해석

각 계층에서 데이터 단위가 달라집니다.

| 계층 | 데이터 단위 |
|:---:|:---:|
| 4 (전송) | 세그먼트 (TCP) / 데이터그램 (UDP) |
| 3 (네트워크) | 패킷 |
| 2 (데이터링크) | 프레임 |
| 1 (물리) | 비트 |

---

## TCP/IP 4계층

OSI는 이론 모델이고, 실제 인터넷은 TCP/IP 4계층으로 동작합니다.

| TCP/IP 계층 | 대응하는 OSI 계층 |
|---|:---:|
| 응용 (Application) | 5 + 6 + 7 |
| 전송 (Transport) | 4 |
| 인터넷 (Internet) | 3 |
| 네트워크 접근 (Network Access) | 1 + 2 |

OSI가 역할을 세분화해서 개념 이해에 유리하다면, TCP/IP는 실용성을 위해 계층을 합쳤습니다.

---

## 정리

| | OSI 7계층 | TCP/IP 4계층 |
|---|---|---|
| 목적 | 개념 표준 (이론) | 실제 인터넷 동작 모델 |
| 계층 수 | 7계층 | 4계층 |
| 사용 목적 | 네트워크 학습·설계 기준 | 실무 프로토콜 기준 |

- 송신 시 캡슐화(헤더 추가), 수신 시 역캡슐화(헤더 제거)
- 각 계층은 독립적으로 동작 → 한 계층의 변경이 다른 계층에 영향을 주지 않음
