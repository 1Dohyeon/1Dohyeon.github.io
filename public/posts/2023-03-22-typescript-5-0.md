# TypeScript 5.0: 무엇이 달라졌나

2023년 3월 TypeScript 5.0이 정식 출시됐습니다. 버전 숫자가 크게 뛴 만큼 굵직한 변화들이 포함됐습니다. 그 중에서 실제로 코드 작성 방식에 영향을 줄 변경 사항들을 정리합니다.

---

## Decorators - 드디어 표준으로

가장 큰 변화입니다. TypeScript는 오래전부터 `experimentalDecorators` 옵션으로 데코레이터를 지원했지만, 이는 TC39 Stage 2 시절의 초안 스펙을 구현한 것이었습니다. 이후 표준 스펙이 Stage 3으로 올라오면서 설계가 크게 바뀌었고, TypeScript 5.0은 이 **Stage 3 표준 데코레이터**를 기본으로 지원합니다.

기존 `experimentalDecorators`와는 동작 방식이 다르기 때문에 호환되지 않습니다. NestJS처럼 데코레이터를 많이 사용하는 프레임워크는 당분간 `experimentalDecorators` 옵션을 유지할 가능성이 높습니다.

```typescript
// Stage 3 표준 데코레이터 예시
function logged(target: any, context: ClassMethodDecoratorContext) {
  return function (this: any, ...args: any[]) {
    console.log(`Calling ${String(context.name)}`);
    return target.call(this, ...args);
  };
}

class Greeter {
  @logged
  greet(name: string) {
    return `Hello, ${name}`;
  }
}
```

## const 타입 파라미터

> 제네릭 함수, 리터럴 타입, 타입 추론 개념이 낯설다면 [TypeScript 핵심 개념 정리](/blog/2023-03-01-typescript-types-and-generics)를 먼저 읽어보시길 권합니다.

**제네릭 함수**는 호출 시점에 타입이 결정되는 함수, **리터럴 타입**은 `string` 대신 `"a"` 같은 구체적인 값 자체가 타입이 되는 것, **타입 추론**은 TypeScript가 타입을 자동으로 파악하는 기능입니다.

기존에는 제네릭 함수에서 타입 추론 시 리터럴 타입 대신 넓은 타입으로 추론되는 경우가 많았는데, TS 5.0에서 `const` 수식어를 붙이면 리터럴 타입 그대로 추론됩니다.

```typescript
// 기존: T가 string[]으로 추론됨 → 반환값이 string
function getFirst<T extends readonly unknown[]>(arr: T): T[0] {
  return arr[0];
}

// TS 5.0: const 추가 → T가 ["a", "b", "c"] 그대로 추론됨 → 반환값이 "a"
function getFirst<const T extends readonly unknown[]>(arr: T): T[0] {
  return arr[0];
}

const result = getFirst(["a", "b", "c"]); // 기존: string → 5.0: "a"
```

## --verbatimModuleSyntax

기존에는 `import type`을 일반 `import`로 작성해도 컴파일러가 알아서 처리해줬습니다. 이 옵션을 켜면 `import type`은 반드시 `import type`으로, 실제 값 import는 `import`로 명확히 구분해서 작성해야 합니다.

```typescript
// verbatimModuleSyntax: true 에서는 이게 오류
import { User } from "./types"; // User가 타입만 있는 경우

// 이렇게 써야 함
import type { User } from "./types";
```

번들러와의 호환성을 높이고, 런타임에 불필요한 import가 남는 문제를 방지합니다.

## 빌드 속도 개선

TypeScript 5.0은 내부 구조를 네임스페이스에서 모듈 방식으로 전환하면서 번들 크기가 줄고 빌드 속도가 **10~25% 빨라졌다고 합니다.** ([공식 발표](https://devblogs.microsoft.com/typescript/announcing-typescript-5-0/) 기준 수치입니다.)

---

## 정리

TS 5.0의 핵심은 **데코레이터 표준화**와 **타입 추론 정밀도 향상**입니다. 당장 마이그레이션이 필요한 breaking change는 거의 없지만, 데코레이터를 사용하는 프로젝트라면 `experimentalDecorators`와 표준 데코레이터의 차이를 미리 파악해두는 것이 좋습니다.
