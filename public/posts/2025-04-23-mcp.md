# MCP(Model Context Protocol): AI 에이전트의 표준화

Anthropic이 공개한 [MCP(Model Context Protocol)](https://modelcontextprotocol.io/)는 LLM이 외부 도구나 데이터 소스와 통신하는 방식을 표준화하는 오픈 프로토콜입니다.

---

## 왜 필요한가

LLM이 파일 읽기, DB 조회, API 호출 같은 작업을 하려면 각 도구마다 별도로 통합 코드를 작성해야 했습니다. A라는 AI 앱에 파일 시스템 연동을 만들었다면, B라는 AI 앱에도 비슷한 코드를 또 만들어야 했습니다.

MCP는 이 통합 방식을 공통 인터페이스로 표준화합니다. USB 포트에 비유하면, USB 이전에는 기기마다 다른 단자가 필요했는데 USB 이후로는 하나의 표준으로 연결할 수 있게 됐듯이, MCP가 LLM 도구 연결의 표준이 되려는 것입니다.

---

## 구조

MCP는 클라이언트-서버 구조입니다.

- **MCP Server**: 외부 도구나 데이터를 노출합니다 (파일 시스템, DB, 외부 API 등)
- **MCP Client**: LLM 애플리케이션 (Claude Desktop, Cursor 등)

MCP Server가 제공할 수 있는 것은 세 가지입니다.

| 종류 | 설명 |
|---|---|
| Resources | LLM이 읽을 수 있는 데이터 (파일, DB 레코드 등) |
| Tools | LLM이 실행할 수 있는 함수 |
| Prompts | 재사용 가능한 프롬프트 템플릿 |

---

## 간단한 MCP Server 예시

```python
from mcp.server import Server
from mcp.types import Tool

server = Server("my-server")

@server.list_tools()
async def list_tools():
    return [Tool(name="get_weather", description="날씨 조회")]

@server.call_tool()
async def call_tool(name, arguments):
    if name == "get_weather":
        return fetch_weather(arguments["city"])
```

이 서버를 MCP를 지원하는 어떤 클라이언트에도 연결할 수 있습니다.

---

## AI 에이전트 개발에서 느낀 것

AI 에이전트를 공부하면서 가장 번거로운 부분 중 하나가 LLM과 외부 도구를 연결하는 코드였습니다. MCP 같은 표준이 확산되면 도구 연동 코드를 매번 새로 짜지 않고, MCP Server를 한 번 만들어두면 여러 클라이언트에서 재사용할 수 있습니다.

아직 생태계가 형성되는 초기 단계지만, Claude Desktop이나 Cursor 같은 주요 도구들이 이미 지원하면서 빠르게 퍼지고 있습니다.
