## 배경

질병은 단백질 구조를 통해 알 수 있고, 단백질 구조는 서열에 의해 정의됩니다. 그렇다면 단백질의 아미노산 서열만으로 관련 질병을 예측할 수 있지 않을까라는 의문에서 시작했습니다. 사용자가 단백질 서열을 입력하면 연관 질병 목록과 발생 확률을 반환하는 시스템을 목표로 했습니다.

## 왜 전통적 네트워크 분석 대신 GCN인가

처음에는 단백질-단백질 상호작용(PPI) 네트워크를 활용하는 방식을 검토했습니다. 하지만 전통적 연결망 분석은 직접 연결된 이웃에만 의존하기 때문에 간접적이고 복잡한 관계(2-hop 이상)를 반영하지 못합니다. 단백질 서열, 기능(GO term) 같은 복합 속성을 네트워크 구조와 함께 통합 학습하기도 어렵고, 데이터가 희소할수록 정확도가 크게 떨어집니다.

**GCN(Graph Convolutional Network)** 은 이 문제를 다르게 접근합니다. 단백질 서열 임베딩 간의 유사성과 관계성을 그래프로 정의하면, 여러 hop에 걸친 간접 관계와 노드 속성을 동시에 학습할 수 있습니다. 단순한 연결 유무가 아니라 "어떻게 연결되어 있고 어떤 생물학적 특성을 가지는가"를 함께 학습하는 구조입니다.

| 전통 연결망의 한계 | GCN의 해결 방식 |
|---|---|
| 직접 연결에만 의존 | k-layer를 통해 다중 hop 이웃 정보를 통합 |
| 속성 통합 어려움 | 서열, 기능 정보 등 노드 feature를 함께 학습 |
| 데이터 결손에 취약 | 구조와 속성 모두 반영한 임베딩으로 강건한 예측 |
| 사전 지식 기반 의존 | 데이터 기반 자동 학습으로 규칙 없이 관계 파악 |

## 시스템 구조

학습 데이터는 단백질-질병-유전자 간 관계 정보(Disease ID, UniProt ID, GO Terms)와 단백질 간 상호작용 강도(combined_score)를 결합해 구성했습니다. 단백질 서열 임베딩을 그래프의 노드로, 상호작용 강도를 엣지 가중치로 설정합니다.

사용자가 아미노산 서열을 입력하면 임베딩 벡터로 변환 후 학습된 GCN 모델을 통과해 연관 질병 목록, 질병 이름, 발생 확률을 반환합니다. 모델 서빙은 FastAPI로 구현했고, 프론트엔드 전체를 React로 개발했습니다.
