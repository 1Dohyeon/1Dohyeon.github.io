# Claude 3 출시: LLM 벤치마크를 어떻게 읽어야 하나

Anthropic이 [Claude 3 패밀리](https://www.anthropic.com/news/claude-3-family)를 출시했습니다. Haiku, Sonnet, Opus 세 가지 모델로 구성되며, 발표 자료에 GPT-4를 능가한다는 벤치마크 수치가 가득했습니다. 그런데 보다 보니 수치보다 수치를 읽는 방법이 더 중요하다는 생각이 들었습니다.

---

## Claude 3 패밀리

세 모델은 속도와 성능의 트레이드오프로 나뉩니다.

| 모델 | 특징 |
|---|---|
| Haiku | 가장 빠르고 저렴, 단순한 작업에 적합 |
| Sonnet | 성능과 비용의 균형 |
| Opus | 가장 강력, 복잡한 추론에 적합 |

---

## 벤치마크 수치를 그대로 믿으면 안 되는 이유

AI 회사들은 자사 모델이 유리하게 나오는 벤치마크를 골라서 발표하는 경향이 있습니다. "GPT-4보다 높다"는 수치도 어떤 태스크에서 측정한 건지 확인해야 합니다.

대표적인 벤치마크들도 한계가 있습니다.

- **MMLU**: 다양한 분야의 객관식 문제. 모델이 학습 데이터에서 본 적 있을 가능성이 있습니다.
- **HumanEval**: 코딩 문제. 실제 복잡한 코드베이스와는 다릅니다.
- **MATH**: 수학 문제. 특정 형식의 문제에만 최적화됐을 수 있습니다.

같은 모델도 프롬프트 방식에 따라 결과가 달라지기 때문에, 발표사가 어떻게 테스트했는지도 중요합니다.

---

## 그러면 어떻게 선택해야 하나

벤치마크보다는 내가 실제로 쓰려는 태스크를 직접 테스트해보는 게 가장 현실적입니다.

공부하면서 쓸 때 기준으로는 이렇게 생각하게 됐습니다.

- 코드 리뷰, 디버깅 → 직접 넣어보고 답변 품질 비교
- 긴 문서 요약 → 컨텍스트 윈도우 크기 확인
- 비용 민감한 경우 → Haiku나 Sonnet으로 시작, 부족하면 Opus

어떤 모델이 최고인지보다 어떤 모델이 내 상황에 맞는지가 더 중요한 질문인 것 같습니다.
