# React 19: 무엇이 달라졌나

[React 19](https://react.dev/blog/2024/12/05/react-19)가 정식 출시됐습니다. 가장 큰 변화는 폼과 비동기 액션 처리 방식이 달라졌다는 점입니다.

---

## Actions: 폼 처리의 변화

비동기 작업을 처리하는 새로운 패턴입니다. 기존에는 폼 제출 시 pending 상태, 에러, 결과를 직접 `useState`로 관리했습니다.

```tsx
// React 18 방식
function Form() {
    const [isPending, setIsPending] = useState(false);
    const [error, setError] = useState(null);

    async function handleSubmit(e) {
        e.preventDefault();
        setIsPending(true);
        try {
            await submitData(formData);
        } catch (err) {
            setError(err);
        } finally {
            setIsPending(false);
        }
    }
}
```

React 19의 `useActionState`를 쓰면 이 패턴이 간결해집니다.

```tsx
// React 19 방식
function Form() {
    const [state, action, isPending] = useActionState(submitData, null);

    return (
        <form action={action}>
            {isPending && <p>제출 중...</p>}
            {state?.error && <p>{state.error}</p>}
            <button type="submit">제출</button>
        </form>
    );
}
```

## 새로운 훅들

**`useFormStatus`**
폼의 제출 상태를 하위 컴포넌트에서 접근할 수 있습니다. 제출 버튼 컴포넌트에서 폼의 pending 상태를 props 없이 읽어올 수 있습니다.

**`useOptimistic`**
서버 응답을 기다리지 않고 UI를 먼저 업데이트합니다. 실패하면 원래 상태로 되돌립니다. 좋아요 버튼 같은 UI에서 반응성을 높일 때 씁니다.

## ref가 props로

`forwardRef`로 감싸지 않아도 `ref`를 props로 직접 받을 수 있게 됐습니다.

```tsx
// 기존: forwardRef 필요
const Input = forwardRef((props, ref) => <input ref={ref} {...props} />);

// React 19: 그냥 props로 받으면 됨
function Input({ ref, ...props }) {
    return <input ref={ref} {...props} />;
}
```

`forwardRef`를 쓸 때마다 코드가 한 단계씩 깊어지는 게 항상 번거로웠는데, 이 변화가 반가웠습니다.

---

## `use()` 훅

Promise나 Context를 컴포넌트 안에서 `use()`로 직접 읽을 수 있습니다. 기존 훅과 달리 조건문 안에서도 호출할 수 있습니다.

```tsx
function Comments({ commentsPromise }) {
    const comments = use(commentsPromise); // Promise를 직접 unwrap
    return comments.map(c => <p key={c.id}>{c.text}</p>);
}
```
