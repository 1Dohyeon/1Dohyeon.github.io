# Attention Is All You Need - Transformer 논문 리뷰

GPT, BERT, Claude 같은 현대 LLM의 기반이 되는 논문입니다. [Attention Is All You Need](https://arxiv.org/abs/1706.03762) (Vaswani et al., 2017). AI 공부를 하다 보니 계속 언급돼서 직접 읽어봤습니다.

---

## 이 논문이 나오기 전

당시 시퀀스 처리의 표준은 RNN(LSTM, GRU)이었습니다. RNN은 토큰을 순서대로 처리하기 때문에 두 가지 문제가 있었습니다.

- **병렬화 불가**: 앞 토큰 처리가 끝나야 다음 토큰을 처리할 수 있어 학습이 느렸습니다.
- **장기 의존성 문제**: 시퀀스가 길어질수록 앞쪽 정보가 사라지는 경향이 있었습니다.

Transformer는 이 두 문제를 모두 해결했습니다.

---

## Attention 메커니즘

핵심 아이디어는 "번역할 때 모든 단어를 균등하게 볼 것이 아니라, 현재 단어와 관련 있는 단어에 더 집중하자"입니다.

각 단어는 세 가지 벡터로 변환됩니다.

- **Query(Q)**: "나는 어떤 정보가 필요한가?"
- **Key(K)**: "나는 어떤 정보를 제공할 수 있는가?"
- **Value(V)**: "나의 실제 정보"

```
Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V
```

(1) Q와 K의 내적으로 각 단어 간 관련도를 계산합니다.
(2) softmax로 확률 분포로 변환합니다 (합이 1).
(3) 그 가중치로 V를 가중합해 최종 표현을 만듭니다.

`√d_k`로 나누는 이유는 차원이 커질수록 내적값이 너무 커져 softmax가 한쪽으로 쏠리는 문제(기울기 소실)를 방지하기 위해서입니다.

---

## Multi-Head Attention

Attention을 한 번만 하지 않고 여러 번(head) 병렬로 수행합니다.

```
head_i = Attention(Q·Wᵢᴼ, K·Wᵢᴷ, V·Wᵢᵛ)
MultiHead = Concat(head_1, ..., head_h) · W_o
```

각 헤드가 다른 관점에서 관계를 학습합니다. 하나의 헤드는 문법적 관계를, 다른 헤드는 의미적 관계를 학습하는 식으로 서로 다른 패턴을 잡아냅니다.

---

## Positional Encoding

Attention은 순서 정보가 없습니다. "나는 밥을 먹었다"와 "밥을 나는 먹었다"를 구분하지 못합니다. 단어가 몇 번째에 위치하는지 알려주기 위해 사인/코사인 함수로 위치 정보를 임베딩에 더합니다.

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

학습하는 파라미터가 아니라 고정된 수식으로 계산합니다.

---

## 왜 혁신적이었나

RNN의 순차 처리를 버리고 전체 시퀀스를 한 번에 병렬 처리할 수 있게 됐습니다. GPU의 병렬 연산 능력을 최대한 활용할 수 있어 학습 속도가 크게 빨라졌고, 훨씬 더 큰 모델을 훈련할 수 있게 됐습니다. 지금의 LLM 시대가 이 논문에서 시작됐다고 해도 과언이 아니라고 생각합니다.
