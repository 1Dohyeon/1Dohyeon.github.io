# GPT-4o: 멀티모달 AI의 일상화

OpenAI가 [GPT-4o](https://openai.com/index/hello-gpt-4o/)를 발표했습니다. "o"는 "omni(전부)"의 약자로, 텍스트·이미지·오디오를 하나의 모델에서 처리합니다.

---

## GPT-4 Turbo와 무엇이 다른가

GPT-4 Turbo도 이미지를 받을 수 있었지만, 내부적으로 모달리티별 처리 파이프라인이 분리돼 있었습니다. GPT-4o는 세 가지를 처음부터 하나의 모델이 네이티브로 처리합니다.

오디오 처리 방식이 특히 달라졌습니다. 기존 음성 기능은 아래처럼 세 단계를 거쳤습니다.

```
음성 입력 → STT(음성→텍스트) → LLM → TTS(텍스트→음성) → 음성 출력
```

GPT-4o는 오디오를 직접 이해하고 직접 생성합니다. 중간 변환 과정에서 사라지던 억양, 감정, 속도 같은 정보가 보존됩니다.

---

## 주요 변화

**속도와 비용**
GPT-4 Turbo 대비 2배 빠르고 절반 가격이라고 [발표됐습니다.](https://openai.com/index/hello-gpt-4o/) API를 공부용으로 써보기에 부담이 줄었습니다.

**이미지 이해**
이미지를 설명하는 수준이 아니라, 그래프의 수치를 읽거나 코드 스크린샷을 분석하는 수준입니다.

**실시간 음성 대화**
응답 지연이 평균 320ms로, 사람과 대화하는 것과 비슷한 수준이라고 합니다.

---

## 흥미로웠던 점

텍스트·이미지·오디오를 각각 다른 모델에서 처리하던 방식에서, 하나의 모델이 모든 입력을 통합해서 이해하는 방향으로 가고 있다는 게 인상적이었습니다. 앞으로 AI 애플리케이션을 만들 때 입력 형식에 덜 구애받을 수 있을 것 같습니다.
