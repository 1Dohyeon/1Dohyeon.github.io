# Vibe Coding: 개발 방식이 바뀌고 있다

Andrej Karpathy(전 OpenAI)가 [Vibe Coding](https://x.com/karpathy/status/1886192184808149383)이라는 개념을 소개했습니다. "코드를 직접 작성하지 않고, 원하는 것을 설명하면 AI가 코드를 만든다"는 방식입니다.

---

## 기존 AI 코딩 도구와 무엇이 다른가

GitHub Copilot이 "코드를 작성하는 중에 다음 줄을 자동완성"이라면, Vibe Coding은 "내가 원하는 기능을 말하면 AI가 파일을 열고, 코드를 작성하고, 실행까지 한다"에 가깝습니다.

Cursor, Claude Code 같은 도구들이 이 방향으로 발전하고 있습니다. 코드 한 줄씩이 아니라 기능 단위로 대화하는 방식입니다.

---

## 직접 써보니

간단한 기능을 빠르게 만들 때는 확실히 빠릅니다. 보일러플레이트 코드나 반복적인 CRUD 로직은 설명 한 번으로 만들어집니다.

다만 한계도 분명히 있습니다.

- AI가 만든 코드를 이해하지 못하면 버그가 생겼을 때 고치기 어렵습니다.
- 전체 구조나 아키텍처는 여전히 직접 판단해야 합니다.
- AI가 잘못된 방향으로 가고 있을 때 알아채려면 기본 지식이 필요합니다.

결국 코드를 타이핑하는 능력보다 **코드를 읽고 판단하는 능력**이 더 중요해지는 것 같습니다.

---

## 느낀 점

개발 방식이 바뀌고 있는 건 분명합니다. 모든 코드를 직접 타이핑하는 시대에서, AI가 생성한 코드를 검토하고 방향을 잡는 역할로 무게중심이 이동하고 있습니다.

어떤 방식이 더 좋다기보다, 코드를 잘 읽고 이해하는 능력은 어떤 방식에서도 필요하다는 생각입니다. 기초가 탄탄할수록 AI를 더 잘 활용할 수 있는 것 같습니다.
