Data Acquisition Preprocessing

Data Objects and Attribute Types

Types of Data Sets

  1. Record
    • 관계형 데이터베이스 튜플(레코드): 데이터베이스의 행과 유사한 형태로 저장되는 데이터.
    • 데이터 매트릭스: 숫자형 행렬이나 교차표와 같은 형태로 표현되는 데이터.
    • 문서 데이터: 텍스트 문서를 단어 빈도 벡터로 표현하는 방법.
    • 트랜잭션 데이터: 구매 이력이나 이벤트 시퀀스와 같은 데이터.
  2. Graph and network
    • World Wide Web: 웹 페이지와 링크로 구성된 네트워크 구조.
    • Social or information networks: 소셜 미디어의 연결망이나 정보의 흐름을 나타내는 네트워크.
    • Molecular(분자구조) Structures: 화학 구조를 나타내는 그래프. 데이터의 예시로 벤젠 분자(C6H6) 구조 등이 있음.
  3. Ordered Data
    • 비디오 데이터(Video data): 이미지의 시퀀스로 구성된 동영상 데이터.
    • 시계열 데이터(Temporal data): 시간 순서에 따른 데이터로, 주식 가격 변동이나 기상 데이터 등이 있음.
    • 순차 데이터(Sequential Data): 거래나 이벤트의 순차적인 흐름을 나타내는 데이터.
    • 유전자 서열 데이터(Genetic sequence data): DNA 서열처럼 순서가 중요한 데이터.
  4. Spatial(공간), image and multimedia
    • 공간 데이터: 지도와 같은 지리적 정보.
    • 이미지 데이터: 사진이나 스캔 이미지와 같은 시각적 데이터.
    • 비디오 데이터: 동영상 형태의 데이터.

Data Engineering Lab Characteristics of Structured Data

  • 차원성 (Dimensionality) – 차원의 저주: 데이터의 차원이 증가함에 따라 분석이 어려워지는 현상
  • 희소성 (Sparsity) – 존재하는 값만 중요: 대부분의 값이 0이거나 비어 있는 경우, 실제로 존재하는 값만 의미
  • 해상도 (Resolution) – 패턴은 스케일에 따라 다름: 데이터가 수집되거나 분석되는 해상도나 스케일에 따른 패턴의 변화
  • 분포 (Distribution) – 중심 경향성 및 분산: 데이터의 중심 경향(평균, 중위수 등)과 분산(범위, 표준 편차 등)과 관련된 특성

Data Objects

  • Data sets는 데이터 객체로 구성
  • Data object는 하나의 개체를 나타냄
  • 예시:
    • 판매 데이터베이스: 고객, 매장, 상품, 판매
    • 의료 데이터베이스: 환자, 치료
    • 대학 데이터베이스: 학생, 교수, 강좌
  • 다른 명칭: 샘플, 예시, 인스턴스, 데이터 포인트, 객체, 튜플
  • 데이터 객체는 속성으로 설명됨
  • 데이터베이스에서 행(row)은 데이터 객체, 열(column)은 속성에 해당됨

Attributes

  • Attribute (또는 차원, 특징, 변수): 데이터 객체의 특성이나 특징을 나타내는 데이터 필드
    • 예: 고객 ID, 이름, 주소

Attribute Types

  • 명목형 (Nominal): 범주, 상태 또는 "사물의 이름"
    • 예: ID 번호, 눈 색깔, 우편번호
  • 순서형 (Ordinal): 값에 의미 있는 순서가 있지만 연속 값 간의 크기는 알 수 없음
    • 예: 순위(예: 감자칩의 맛을 1-10으로 평가), 등급, 키(높음, 중간, 낮음)
  • 수치형 (Numeric): 정량적 속성
    • 구간 척도 (Interval-scaled): 예) 달력 날짜, 섭씨 또는 화씨 온도
    • 비율 척도 (Ratio-scaled): 예) 길이, 시간, 개수
  • 이진형 (Binary): 두 가지 상태만 가지는 명목형 속성 (0과 1)
    • 대칭적 이진형 (Symmetric binary): 두 결과가 동일한 중요성
      • 예: 성별
    • 비대칭적 이진형 (Asymmetric binary): 결과의 중요성이 다름
      • 예: 의료 테스트 (양성 vs 음성)
      • 관례: 가장 중요한 결과에 1을 할당 (예: HIV 양성)

Numeric Attribute Types

  • 수량 (Quantity): 정수 또는 실수 값
  • 구간 척도 (Interval)
    • 동일한 크기의 단위로 측정
    • 값에 순서가 있음
      • 예: 섭씨 또는 화씨 온도, 달력 날짜
    • 절대적인 0점이 없음
  • 비율 척도 (Ratio)
    • 고유한 0점이 존재
    • 값의 크기를 측정 단위의 배수로 표현 가능 (예: 10 K˚는 5 K˚의 두 배)
    • 예: 켈빈 온도, 길이, 개수, 화폐량

Discrete vs. Continuous Attributes

  • 이산형 속성 (Discrete Attribute)
    • 유한하거나 셀 수 있는 무한한 값의 집합만 가짐
      • 예: 우편번호, 직업, 문서의 단어 집합
    • 때로는 정수형 변수로 표현
    • 참고: 이진형 속성은 이산형 속성의 특별한 경우
  • 연속형 속성 (Continuous Attribute)
    • 실수 값을 가지는 속성
      • 예: 온도, 키, 몸무게
    • 실제로는 실수 값을 유한한 자릿수로 측정 및 표현
    • 일반적으로 부동소수점 변수로 표현

Basic Statistical Descriptions of Data

Basic Statistical Descriptions of Data(데이터의 기본 통계적 설명)

  • 동기 (Motivation): 데이터의 중심 경향성, 변동 및 분포를 더 잘 이해하기 위함
  • 데이터 분산 특성(Data dispersion characteristics): 중위수, 최대값, 최소값, 분위수, 이상치, 분산 등
  • 수치적 차원(Numerical dimensions)은 정렬된 구간에 대응
    • 데이터 분산: 다양한 정밀도로 분석
    • 박스플롯 또는 분위수 분석을 통해 정렬된 구간 분석
  • 계산된 측정치에 대한 분산 분석(Dispersion analysis on computed measures): 측정치를 수치적 차원으로 변환
    • 변환된 큐브에서 박스플롯 또는 분위수 분석

Measuring the Central Tendency(중심 경향성 측정)

  • 평균 (Mean): 대수적 측정치로 표본과 모집단에 따라 다름
    • n: 표본 크기, N: 모집단 크기
    • 가중 산술 평균 (Weighted arithmetic mean)
    • 절단 평균 (Trimmed mean): 극단적인 값 제거
  • 최빈값 (Mode)
    • 데이터에서 가장 자주 발생하는 값
    • 단봉형(Unimodal), 이봉형(Bimodal), 삼봉형(Trimodal), multimodal(최빈값이 여러개)
    • 경험적 공식(Empirical formula): mean - mode = 3 x (mean - median(중앙값))
  • Median(중앙값)
    • Middle value(총 value가 짝수라면 2개의 평균)
    • frequency를 다 더하면 2749, 2749/2 = 1374.5 = n/2 보다 큰 1500 frequency 내부에 median이 있다고 유추 가능
    • median = 21 + ( (1374.5 - 505)/1500 ) x 29
    • 505는 이전 frequency의 누적값
    • 21은 21-50의 첫 value
    • 29는 21-50의 range

Symmetric vs. Skewed Data

  • 대칭형(Symmetric) 데이터: 평균, 중위수, 최빈값이 동일한 위치에 있음
  • 양의 왜곡 (Positively skewed): 오른쪽 꼬리가 긴 분포. 최빈값 < 중위수 < 평균
  • 음의 왜곡 (Negatively skewed): 왼쪽 꼬리가 긴 분포. 평균 < 중위수 < 최빈값

Measuring the Dispersion of Data

  • 사분위수, 이상치, 박스플롯
    • 사분위수 (Quartiles): 데이터를 크기 순서대로 나열했을 때, 전체를 4등분하는 위치의 값
      • Q1: 하위 25%의 값, 즉 데이터의 25% 지점
      • Q3: 상위 25%에 해당하는 값, 즉 데이터의 75% 지점
    • 사분위수 범위 (Inter-quartile range, IQR): Q3에서 Q1을 뺀 값으로, 데이터의 중간 50%가 어느 정도의 범위에 퍼져 있는지를 나타냄
    • 5가지 요약 (Five-number summary): 데이터를 간단하게 요약하는 다섯 가지 숫자
      • 최소값, Q1, 중위수(중간값), Q3, 최대값
    • 박스플롯 (Boxplot): 데이터를 시각적으로 표현한 그림
      • 박스의 양쪽 끝은 Q1과 Q3를 나타내고, 박스 안의 선은 중위수를 표시
      • 박스 바깥쪽으로 뻗은 선을 "수염"이라고 부르며, 데이터의 전체 범위를 보여줌
      • 수염 바깥에 위치한 점들은 이상치로 간주
    • 이상치 (Outlier): 데이터의 나머지 값들과 비교했을 때 눈에 띄게 크거나 작은 값
      • 일반적으로 Q1보다 1.5 x IQR만큼 작거나 Q3보다 1.5 x IQR만큼 큰 값을 이상치로 간주
  • 분산과 표준 편차
    • 분산 (Variance): 데이터가 평균에서 얼마나 떨어져 있는지 측정한 값
      • 값들이 평균 주변에 얼마나 퍼져 있는지를 나타냄
    • 표준 편차 (Standard Deviation): 분산의 제곱근
      • 데이터가 평균에서 얼마나 떨어져 있는지 쉽게 이해할 수 있는 값으로, 분산의 단위를 원래 데이터의 단위로 되돌림
      • 표준 편차가 크면 데이터가 평균 주변에서 넓게 퍼져 있고, 작으면 평균에 가까이 모여 있는 것임

Boxplot Analysis

  • 분포의 5가지 요약은 데이터를 간단하게 나타내는 다섯 가지 숫자:
    • 최소값: 가장 작은 값
    • Q1 (1사분위수): 데이터의 하위 25%에 해당하는 값
    • 중위수 (Median): 데이터를 반으로 나눴을 때 그 중간에 있는 값
    • Q3 (3사분위수): 데이터의 상위 25% 지점 값
    • 최대값: 가장 큰 값
  • 박스플롯은 이 다섯 가지 요약 값을 그림으로 나타낸 것:
    • 데이터를 한눈에 볼 수 있도록 박스로 표현
    • 박스의 양쪽 끝은 Q1과 Q3를 나타내며, 이 범위는 데이터의 중간 50%가 어디에 있는지를 보여줌
    • 박스 안의 선은 중위수(데이터의 중간 위치)를 표시
    • 박스의 바깥쪽에 뻗어 있는 선은 "수염"으로, 최소값부터 최대값까지의 범위를 나타냄
    • 데이터 중 눈에 띄게 크거나 작은 값은 박스 바깥에 점으로 표시되어 이상치로 구분

Graphic Displays of Basic Statistical Descriptions

  • 박스플롯 (Boxplot): 데이터의 중요한 다섯 가지 숫자 (최소값, 1사분위수, 중위수, 3사분위수, 최대값)를 한눈에 볼 수 있는 그래프
  • 히스토그램 (Histogram): 데이터를 여러 구간으로 나누고, 각 구간에 해당하는 데이터의 개수를 세어 보여주는 그래프
    • x축은 값의 범위, y축은 그 값에 해당하는 데이터의 개수
  • 분위수 플롯 (Quantile plot): 데이터가 얼마나 퍼져 있는지 보여주는 그래프
    • 각 데이터 값을 얼마나 많은 데이터가 차지하고 있는지를 퍼센트로 나타냄
  • q-q 플롯 (Quantile-quantile plot): 두 데이터 분포를 비교하는 그래프
    • 한 분포의 분위수를 다른 분포의 분위수와 나란히 그려, 두 분포가 얼마나 비슷한지 확인
  • 산점도 (Scatter plot): 두 가지 데이터 간의 관계를 점으로 표시한 그래프
    • 예를 들어 키와 몸무게의 관계를 보여줄 때 각 사람의 키와 몸무게를 한 점으로 표시
Histogram Analysis
  • 히스토그램 (Histogram): 막대 그래프로 각 값의 빈도를 나타내는 그래프
  • 여러 범주에 해당하는 데이터가 각각 얼마나 많은지 보여줌
  • 막대그래프와의 차이점: 막대그래프는 막대의 높이가 값을 나타내지만, 히스토그램은 막대의 면적이 값을 나타냄. 특히, 범주의 폭이 일정하지 않을 때 중요
  • 범주는 일반적으로 변수의 서로 겹치지 않는 구간으로 나누며, 각 막대는 인접해 있어야 함
Quantile Plot
  • 모든 데이터를 보여주어 사용자가 전체적인 경향과 특이한 점을 파악할 수 있게 함
  • 분위수 정보 표시: 데이터 `xi`를 오름차순으로 정렬할 때, `fi`는 데이터의 약 `100 X fi%`가 `xi` 이하임을 나타냄
Data Engineering Lab Quantile-Quantile (Q-Q) Plot
  • 역할: 한 분포의 분위수를 다른 분포의 분위수와 비교하여 그린 그래프
  • 목적: 두 분포 간에 차이가 있는지, 또는 한 분포가 다른 분포에 비해 어떻게 이동(shift)하는지 확인
  • 예시: 지점 1과 지점 2에서 판매된 상품의 단가를 비교하는 경우, Q-Q 플롯을 통해 지점 1의 단가가 지점 2보다 대체로 낮다는 것을 확인

쉽게 말해, Q-Q 플롯은 두 데이터 세트가 얼마나 비슷한지 또는 어떻게 다른지를 비교하는 그래프.

Scatter plot
  • 역할: 두 변수를 가진 데이터(bivariate data)를 처음 살펴보고, 데이터의 군집(클러스터)이나 이상치(특이한 값)를 확인
  • 표현 방식: 각 값의 쌍을 좌표로 취급하여 평면 위에 점으로 표시

쉽게 말해, 산점도는 두 변수 간의 관계를 점으로 나타내어 시각적으로 확인하는 그래프.

Positively and Negatively Correlated Data
  • 양의 상관관계 (Positively correlated): 한 값이 증가할 때 다른 값도 함께 증가하는 경향이 있는 데이터 (왼쪽 부분)
  • 음의 상관관계 (Negatively correlated): 한 값이 증가할 때 다른 값은 감소하는 경향이 있는 데이터 (오른쪽 부분)

MEASURING DATA SIMILARITY AND DISSIMILARITY

Similarity and Dissimilarity

  • 유사성 (Similarity)
    • 두 데이터 객체가 얼마나 유사한지를 나타내는 수치적 척도
    • 값이 높을수록 두 객체가 더 유사함
    • 주로 0에서 1 사이의 범위를 가짐
    • 예를 들어, 두 사람의 키가 비슷하면 유사성이 높음 (1에 가까움).
  • 비유사성 (Dissimilarity, 예: 거리)
    • 두 데이터 객체가 얼마나 다른지를 나타내는 수치적 척도
    • 값이 낮을수록 두 객체가 더 유사함
    • 최소 비유사성은 보통 0이고, 최대값은 상황에 따라 다름
    • 두 사람의 키 차이가 크다면 비유사성이 높음 (0에서 멀어짐).
  • 근접도 (Proximity) 는 유사성 또는 비유사성을 의미

Proximity Measure for Binary Attributes

  • symmetric binary variables에서 각 변수의 측정 거리:
    • d(i, j ) = (r+s) / (q+r+s+t)
  • asymmetric binary variables에서 각 변수의 측정 거리:
    • d(i, j ) = (r+s) / (q+r+s)
  • d(i, j)는 각 변수들의 거리이기에 유사성을 구할 때는 1에서 d(i, j)를 빼주면 됨.
    • sim(i, j) = 1 - d(i, j)
  • r: i=1, j=0
  • s: i=0, j=1
  • q: 둘 다 1
  • t: 둘 다 0

위 문제에서는 P가 1, N이 0이라고 가정

즉, d(Jack, Mary) = (r+s=0+1=1)/(q+r+s=2+0+1) = 0.33
d(Jack, Jim) = (r+s=1+1=2)/(q+r+s=1+1+1=3) = 0.66
d(Mary, Jim) = (r+s=2+1=3)/(q+r+s=1+2+1=4) = 0.75

따라서 d(i, j)가 가장 작은 값은 Jack과 Mary이다. Jack과 Mary의 증상은 매우 가깝다. 따라서 유사도 sim(i, j) = 1 - d(i, j) = 0.67로 가장 큰 값을 가진다.

Jack과 Mary가 유사한 질병을 가질 확률이 크다.

가장 유명한 거리 측정 방식은 민코프스키 거리(Minkowski Distance)이다.

벡터 공간 모델(Vector Space Model)

벡터 공간 모델은 문서가 벡터로 표현된다는 가정하에 문서 간의 유사성을 측정하는 방법이다. 각 단어가 벡터의 차원이 되고, 문서는 해당 단어의 빈도수로 표현되는 벡터가 된다.

벡터 공간 모델 개념

  1. 문서(Document)는 벡터이다: 각 단어가 벡터의 차원이 되며, 각 차원의 값은 문서에서 해당 단어의 빈도수로 나타낸다.
  2. 가정:
    • 예시로 단어가 3개(AOA, 블랙핑크, 트와이스)만 존재한다고 가정하고, 데이터베이스에 3개의 문서만 존재한다고 가정한다.
      • 문서 1 (D1): "AOA 3회, 블랙핑크 2회, 트와이스 1회"
      • 문서 2 (D2): "AOA 2회, 트와이스 2회"
      • 문서 3 (D3): "블랙핑크 2회, 트와이스 1회, AOA 2회"
    • 이 세 문서를 각각 벡터로 표현하면 다음과 같다:
      • D1 = (3, 2, 1)
      • D2 = (2, 0, 2)
      • D3 = (2, 2, 1)
  3. 코사인 유사도(Cosine Similarity): 벡터 간의 유사도를 측정하는 방법으로, 코사인 유사도가 주로 사용된다.
    • 두 벡터 사이의 각도를 측정하여 유사도를 계산한다.
    • 두 벡터 d1과 d2의 코사인 유사도는 다음과 같다
    • 이때 오른쪽 식의 분자는 내적이다.
      • 내적 공식: x ⋅ y = ||x|| ||y|| cosθ
      • ∣∣d1∣∣과 ∣∣d2∣∣는 각각의 벡터의 길이(크기)이다.
    • 코사인 값이 1에 가까울수록 두 벡터는 유사하고, 0이면 전혀 유사하지 않다(직각). 코사인 값이 -1이면 완전히 반대의 방향을 나타낸다.

코사인 유사도의 계산 예시

문서 1과 문서 2 사이의 코사인 유사도를 계산한다고 가정하면, 문서 1의 벡터 d1=(3,2,1), 문서 2의 벡터 d2=(2,0,2)이다.

  • d1⋅d2=3×2+2×0+1×2=8
  • ∣∣d1∣∣=3^2+2^2+1^2=14
  • ∣∣d2∣∣=2^2+0^2+2^2=8
  • 따라서, cos(d1,d2)=8/(14×8)≈0.76

이렇게 코사인 유사도를 통해 문서 간의 유사도를 수치화할 수 있다. 이는 정보 검색, 문서 분류 등 다양한 분야에서 문서 간의 유사성 측정에 활용된다.
