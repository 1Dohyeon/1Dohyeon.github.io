Dimensionality Reduction

1. Dimensionality Reduction

개념

Dimensionality Reduction(차원 축소)란 dimension reduction로도 불리며, 고차원의 데이터를 더 낮은 차원의 데이터로 변환원의 데이터를 더 낮은 차원의 데이터로 변환하는 과정이다.

이 저차원 표현이 원래 데이터의 중요한 속성을 유지하며, 이상적으로는 데이터의 본질적인 차원에 가깝게 만드는 것을 목표로 한다.

데이터 세트의 본질적인 차원은 데이터를 최소한으로 표현하는 데 필요한 변수의 수로 생각할 수 있다.

차원축소를 하게되면, 원래 데이터의 변동성 일부를 잃게 된다. 보통 유지하는 특징이나 구성 요소의 수에 따라서 1%에서 15% 정도의 변동성이 사라진다.

Advantages of Dimensionality Reduction

  • 더 적은 훈련 시간과 계산 자원을 사용하며, 머신러닝 알고리즘의 전체 성능을 향상시킨다.
    • 과적합(overfitting) 문제를 방지한다.
    • 고차원 데이터셋에서는 대부분의 데이터 포인트가 서로 멀리 떨어져 있어, 알고리즘이 고차원 데이터를 효과적으로 훈련시키지 못한다. (curse of dimensionality : 차원의 저주)
    • 데이터 압축을 제공한다.
  • 데이터 시각화에 매우 유용하다.
    • 데이터를 2D 또는 3D 그래프로 쉽게 표현할 수 있다.
  • 해석 가능성을 제공한다.
  • Multicollinearity(다중공선성) 문제를 해결한다.

    Multicollinearity이란?

    • 두 개 이상의 예측 변수들 간의 높은 상관관계이다.
    • 예: 사람의 키와 몸무게.
    • 문제: 각 예측 변수가 반응 변수에 미치는 영향을 정확히 구분하기 어렵게 만든다.

    이러한 높은 상관관계를 가진 변수를 비상관성 변수 집합으로 결합한다.

  • 데이터에서 노이즈를 제거한다.
    • 가장 중요한 특징을 남기고 중복된 특징을 제거하여, 차원 축소는 데이터에서 노이즈를 제거한다.

2. Methods for Dimensionality Reduction

특징 선택 (Feature Selection)

  • 데이터셋에서 가장 중요한 특징을 유지하고 중복된 특징을 제거한다.
  • 입력 변수를 줄여, 목표 변수를 예측하는 데 가장 유용하다고 생각되는 변수만 남긴다.
  • 특징 세트에 변환이 적용되지 않는다.

특징 결합 (Combination of Features (Feature Extraction))

  • 원래 값 대신 다른 값을 포함하는 새로운 특징 세트를 찾는다.
  • 특징 투영(Feature projection).
  • 선형/비선형 방식.

Feature Selection vs Feature Extraction

  • 두 방법 모두 차원 축소로 통칭된다.
  • 특징 선택(Selection): 주어진 𝑑개의 특징에서 크기 𝑚인 최적의 하위 집합을 선택한다.
  • 특징 추출(Extraction): 주어진 𝑑개의 특징(세트 𝑌)에서 선형 또는 비선형 결합을 통해 𝑚개의 새로운 특징(세트 𝑋)을 추출한다.
    • 선형 특징 추출: 𝑋 = 𝑇𝑌, 여기서 𝑇는 𝑚 ∗ 𝑑 행렬이다.
    • 비선형 특징 추출: 𝑋 = 𝑓(𝑌)
      • 예: 오토인코더(Auto Encoder)
  • 선택/추출의 기준: 분류 정확도를 향상시키거나 유지하고, 분류기의 복잡성을 단순화한다.

3. Feature Selection

머신러닝과 통계에서 특징 선택은 변수 선택, 속성 선택 또는 변수 하위 집합 선택이라고도 하며, 모델 구성에 사용할 관련 특징(변수, 예측 변수)의 하위 집합을 선택하는 과정이다.

즉, 중요한 특징을 고르고, 불필요하거나 중복된 특징을 제거함으로써 모델의 성능을 높이고, 계산 자원을 절약할 수 있다.

Filter Methods

  • 주로 전처리 과정에서 쓰인다.
  • 통계적 측정치를 기반으로 특징을 선택한다.
  • 데이터를 분석하기 전, 통계적 방법으로 가장 의미 있는 특징을 선택하는 것이다. 이 방법은 빠르고 계산 비용이 적지만, 모델의 예측 성능을 크게 높이는 데는 제한적일 수 있다. 그래서 주로 데이터를 준비할 때 많이 사용된다.
    • 통계적 점수를 기반으로 특징을 선택한다.
    • Filter Methods에서의 특징 선택 과정은 보통 다음과 같다 :
    • set of all features -> selecting the best subset -> learning algorithm -> performance
  • 정확도는 낮지만 계산이 빠름, 따라서 다음 상황에서 사용된다 :
    • Correlation Coefficient(상관 계수) : 변수 간의 상관관계를 측정해 중요한 변수를 고른다.
    • Chi-square Test(카이제곱 검정) : 범주형 변수 간의 독립성을 테스트해 특징을 선택한다.
    • Variance Threshold(분산 임계값) : 분산이 작은 변수를 제거한다. 이는 그 변수가 큰 변화를 주지 않기 때문이다.

Wrapper Methods

  • 특징 선택을 탐색 문제로 간주한다.
  • 가능한 모든 특징 조합의 하위 집합 공간을 탐색하는 방법을 요구하며, 그 하위 집합을 사용해 학습하고 분류기를 평가하여 그 품질을 평가한다.
    • 우리가 주어진 데이터셋에 맞추고자 하는 특정 머신러닝 알고리즘에 기반한다.
    • 그리디 탐색 방법으로, 특징들의 모든 가능한 조합을 평가 기준에 따라 평가한다.
  • set of all features -> Generate a subset -> learning algorithm -> performance 순서지만, Generate a subset -> learning algorithm 이 과정을 반복한다.
  • 이 방법은 모델을 실제로 여러 번 실행해보면서 가장 좋은 성능을 내는 변수 조합을 찾는 것이다. 단점은 시간이 오래 걸리지만, 변수 간의 상호작용을 잘 반영할 수 있다는 점에서 유리하다.
    • 그렇다면 어떻게 성능을 평가한다는 것일까? 성능 평가 기준 (Performance Criteria)
    • Model evaluation(모델 평가)
  • 다음 상황에서 사용된다 :
    • Exhaustive Feature Selection(전수 탐색) : 가능한 모든 변수 조합을 시도해 최적의 조합을 찾는다. 즉, 모든 경우의 수를 하나씩 다 확인하는 방식이다.
    • Forward Feature Selection(전진 특징 선택) : 변수를 하나씩 추가하면서 모델 성능이 좋아지는지를 확인한다. 즉, 처음엔 아무 변수도 없지만, 가장 중요한 것부터 하나씩 더해가는 방식이다. (그리디 방식)
    • Backward Feature Elimination : 처음엔 모든 변수를 포함한 후, 하나씩 제거하면서 성능에 미치는 영향을 본다. 즉, 다 가지고 시작해서 덜 중요한 것들을 차례로 빼는 방식이다.

Embedded Methods

  • 필터 방법과 래퍼 방법의 장점을 결합하면서 특징 간 상호작용을 고려하고, 계산 비용이 낮다.
  • 필터 방법처럼 빠르지만, 래퍼 방법처럼 변수 간의 상호작용을 고려해준다. 따라서 빠른 계산 속도와 높은 성능을 동시에 얻을 수 있는 방법이다. 주로 모델 훈련 과정에서 자동으로 중요한 변수를 찾아낸다.
  • 다음 상황에서 사용된다 :
    • Lasso Regularization : 변수들 간의 중요도를 평가하면서 불필요한 변수들의 가중치를 0으로 만들어 제거한다. 쉽게 말하면, 덜 중요한 변수를 자동으로 걸러내는 방식이다.
    • Random Forest Feature Importance : 랜덤 포레스트 모델에서 변수의 중요도를 평가해, 가장 중요한 변수를 자동으로 선택한다. 즉, 나무를 여러 개 만드는 과정에서 어떤 변수가 가장 자주 쓰이는지에 따라 중요한 변수들을 정리해준다.

4. (Preliminary) Geometric interpretation of Linear combination

선형 결합의 개념

벡터 공간에서 선형 결합은 여러 벡터를 특정 가중치(스칼라)를 사용하여 결합하는 과정을 말한다. 주어진 벡터들을 스칼라 값으로 곱하고 더해서 새로운 벡터를 만드는 것이다.

예를 들어, 두 벡터 v1v_1와 v2v_2가 있을 때, a와 b가 스칼라(숫자)라고 하면: v = av1 + bv2.

이렇게 만들어진 벡터 v가 v1와 v2의 선형 결합이다.

기하학적 해석

기하학적으로 해석하면, 두 벡터 v1와 v2의 선형 결합은 이 두 벡터를 시작점으로 하는 평면이나 직선 상의 모든 벡터를 나타낸다.

즉, 여러 벡터의 선형 결합을 통해 해당 벡터들이 펼칠 수 있는 공간을 정의할 수 있는 것이다.

예를 들어, 2차원 평면에서 두 벡터 v1와 v2가 서로 평행하지 않으면, 그 두 벡터의 모든 선형 결합은 2차원 평면 전체를 표현할 수 있게 된다.

스팬(Span)

스팬은 주어진 벡터들의 모든 선형 결합으로 만들어지는 공간을 의미한다. 주어진 벡터 v1와 v2가 있다고 할 때, 이 두 벡터의 스팬은 av1 + bv2와 같이 a와 b가 모든 실수 값을 가질 때 만들어지는 모든 벡터의 집합이다.

  • 만약 v1와 v2가 2차원 공간의 벡터라면, 이들의 스팬은 해당 2차원 공간 전체가 된다.
  • 반대로 v1와 v2가 같은 직선 위에 놓여 있는 경우라면, 이들의 스팬은 그 직선 자체만을 포함한다.

기저(Basis)

기저란 벡터 공간을 표현하는 최소한의 벡터 집합이다. 벡터 공간을 이루는 벡터 중 선형적으로 독립적인 최소한의 벡터들을 선택하여 그 공간을 표현할 수 있다면, 이 벡터들의 집합을 기저라고 한다.

예를 들어, 2차원 공간에서 벡터 i=(1, 0)와 j=(0,1)는 이 공간을 표현하는 기저이다. 이 두 벡터를 선형 결합하면 2차원 공간의 모든 벡터를 표현할 수 있다.

기저는 항상 벡터 공간의 차원과 같은 수의 벡터를 포함한다. 3차원 공간의 기저는 세 개의 벡터를 포함하며, 각각이 서로 선형적으로 독립적이다. 이는 기저 벡터들이 서로 평행하지 않고 독립적인 방향을 갖는다는 의미이다.

차원 축소의 목표

차원 축소의 목표는 데이터의 축을 찾아내어 표현하는 것이다. 데이터가 고차원 공간에 분포되어 있을 때, 이 데이터의 변동성을 가장 잘 설명하는 축을 찾고 해당 축을 따라 데이터를 저차원으로 표현하는 것이 차원 축소의 핵심이다.

이러한 과정을 통해 일부 정보 손실이 있을 수 있지만, 데이터의 핵심 구조와 중요한 패턴을 유지하면서 더 낮은 차원으로 데이터를 압축할 수 있다.

결국, 차원 축소는 원래 고차원 데이터가 가진 복잡성을 줄이고, 중요한 정보만을 담아 효율적으로 표현하는 것이 목표이다.

차원 축소의 방법은 다음과 같다:

  • PCA(Principal component analysis): 데이터의 분산이 가장 큰 방향을 찾는 것으로, 데이터를 몇 가지 주성분을 기반으로 표현함으로써 차원을 축소.
  • SVD(Singular Value Decomposition): 모든 행렬을 세 개의 행렬 U, Σ, VT의 곱으로 분해하는 방법. 데이터의 근사치를 제공하며, 가장 큰 특이값만을 사용해 데이터의 주요한 정보만 남기고 나머지를 버림으로써 차원을 축소.
